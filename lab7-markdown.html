Variable respuesta categórica: precio_cat

Se definen tres categorías usando los percentiles 33 y 67:

  • Económico: precio ≤ percentil 33 (≤ $141)
  • Medio: percentil 33 < precio ≤ percentil 67 ($141 – $270)
  • Caro: precio > percentil 67 (> $270)
# Calcular umbrales con terciles
q33 <- quantile(listings$price_num, 0.33)
q67 <- quantile(listings$price_num, 0.67)

cat("Umbral económico / medio:", q33, "\n")
## Umbral económico / medio: 141
cat("Umbral medio / caro:     ", q67, "\n")
## Umbral medio / caro:      270
# Crear variable categórica
listings <- listings %>%
  mutate(precio_cat = case_when(
    price_num <= q33 ~ "economico",
    price_num <= q67 ~ "medio",
    TRUE             ~ "caro"
  ))

Interpretación global:

  • El Pseudo R² de McFadden entre 0.2 y 0.4 indica un buen ajuste para modelos de regresión logística.
  • El AUC-ROC mide la capacidad discriminativa del modelo: valores cercanos a 1 indican excelente separación entre clases.
  • La exactitud, sensibilidad y especificidad en el conjunto de prueba permiten evaluar si el modelo generaliza bien a datos no vistos.
  • Si la exactitud en prueba es cercana a la de entrenamiento (obtenida por validación cruzada), el modelo no presenta sobreajuste (overfitting).

Lectura de los resultados:

  • AUC-ROC: indica la probabilidad de que el modelo asigne una probabilidad mayor a un Airbnb caro que a uno no-caro. Valores ≥ 0.80 indican buena discriminación.
  • Kappa de Cohen: corrige la exactitud por la probabilidad de acuerdo al azar. Valores > 0.60 reflejan acuerdo sustancial.
  • F1-Score: relevante cuando las clases son desbalanceadas; combina precisión y recall en una sola métrica.
  • Exactitud Balanceada: útil cuando hay desbalance de clases; pondera igual ambas clases.
  • Si la sensibilidad es notablemente más baja que la especificidad (o viceversa), el umbral de 0.5 puede no ser óptimo para el problema.

6. Sobreajuste (Overfitting) y Curvas de Aprendizaje

6.1 Comparación directa: error de entrenamiento vs. error de prueba

Para diagnosticar overfitting se calcula el error de clasificación (1 − Accuracy) en:

  1. El conjunto de entrenamiento (predicciones in-sample).
  2. El conjunto de prueba (predicciones out-of-sample).
# --- Error en entrenamiento ---
probs_train <- predict(modelo_cv, newdata = train_cv, type = "prob")[, "Caro"]
preds_train <- factor(ifelse(probs_train >= 0.5, "Caro", "NoCaro"),
                      levels = c("NoCaro", "Caro"))

cm_train <- confusionMatrix(preds_train, train_cv$es_caro_f, positive = "Caro")
error_train <- 1 - as.numeric(cm_train$overall["Accuracy"])

# --- Error en prueba ---
error_test  <- 1 - as.numeric(acc5)

Situación Error entrenamiento Error prueba Brecha

Overfitting Muy bajo Mucho mayor Grande
Underfitting Alto Alto Pequeña

6.2 Curvas de aprendizaje

Las curvas de aprendizaje muestran cómo evolucionan los errores a medida que se incrementa el tamaño del conjunto de entrenamiento. Si el modelo generaliza bien, ambas curvas convergen; si hay overfitting, la brecha persiste con conjuntos grandes.

set.seed(123)

# Proporciones del conjunto de entrenamiento a evaluar
proporciones <- c(0.05, 0.10, 0.15, 0.20, 0.30, 0.40,
                  0.50, 0.60, 0.70, 0.80, 0.90, 1.00)

n_train <- nrow(train_m)

# Datos de prueba en formato glm (matriz de diseño)
X_test  <- model.matrix(es_caro ~ accommodates + bedrooms + bathrooms + beds +
                           room_type + host_is_superhost + instant_bookable +
                           minimum_nights + number_of_reviews + availability_365,
                         data = test_m)
y_test  <- test_m$es_caro

resultados_lc <- data.frame()

for (prop in proporciones) {
  idx_sub  <- sample(seq_len(n_train), size = floor(prop * n_train))
  sub_data <- train_m[idx_sub, ]

  # Ajustar modelo logístico
  mod_sub <- glm(es_caro ~ accommodates + bedrooms + bathrooms + beds +
                   room_type + host_is_superhost + instant_bookable +
                   minimum_nights + number_of_reviews + availability_365,
                 data   = sub_data,
                 family = binomial(link = "logit"))

  # Error en subconjunto de entrenamiento
  p_tr  <- predict(mod_sub, newdata = sub_data, type = "response")
  e_tr  <- mean(ifelse(p_tr >= 0.5, 1, 0) != sub_data$es_caro)

  # Error en conjunto de prueba completo
  p_te  <- predict(mod_sub, newdata = test_m, type = "response")
  e_te  <- mean(ifelse(p_te >= 0.5, 1, 0) != y_test)

  resultados_lc <- rbind(resultados_lc,
                         data.frame(
                           prop       = prop,
                           n_obs      = floor(prop * n_train),
                           error_train = e_tr,
                           error_test  = e_te
                         ))
}

cat(sprintf("Brecha final (test - train)            : %.4f\n", brecha_final))
## Brecha final (test - train)            : 0.0048
if (brecha_final < 0.02) {
  cat("\nDiagnóstico: SIN sobreajuste significativo.\n")
  cat("Ambos errores son similares; el modelo generaliza bien.\n")
} else if (brecha_final < 0.05) {
  cat("\nDiagnóstico: sobreajuste LEVE.\n")
  cat("La diferencia entre errores existe pero es pequeña.\n")
} else {
  cat("\nDiagnóstico: sobreajuste MODERADO/SEVERO.\n")
  cat("El error de prueba supera notablemente al de entrenamiento.\n")
}

## Ambos errores son similares; el modelo generaliza bien.

Interpretación de las curvas:

  • Si la curva de entrenamiento comienza muy baja y sube al crecer el dataset → el modelo memoriza los datos con pocas muestras (overfitting en muestras pequeñas), pero generaliza mejor con más datos.
  • Si la curva de prueba desciende al crecer el dataset y se aproxima a la de entrenamiento → el modelo se beneficia de más datos y tiende a un buen ajuste.
  • Si ambas curvas convergen y se estabilizan en una zona baja → el modelo ha alcanzado su capacidad óptima sin overfitting.
  • Si la brecha entre ambas curvas persiste incluso con el 100 % de los datos de entrenamiento → existe overfitting estructural y convendría regularización (e.g., Ridge/LASSO) o reducción de variables.
